Defines the most important heading.

  • h1 means first heading, it's the first heading to a body paragraph.
  • In appearance, its font-size is much bigger than the other headings.

<h1>This is an eaxmple of how it looks in HTML</h1>

h2 back to top

Defines the second most important heading.

  • h2 means second heading, it comes after the first heading and its content.
  • The font-size gets a bit smaller.

<h2>This is an example of how it looks like in HTML</h2>

h3 back to top

Defines the third most important heading.

  • h3 means third heading, it comes after the second heading and its content.
  • The font-size gets a bit smaller.

<h3>This is an example of how it looks like in HTML</h3>

h4 back to top

Defines the fourth most important heading.

  • h4 means fourth heading, it comes after the third heading and its content.
  • The font-size gets smaller and smaller.

<h4>This is an example of how it looks like in HTML</h4>

h5 back to top

Defines the fifth most important heading.

  • h5 means fifth heading, it comes after the fourth heading and its content.
  • The font-size gets smaller.

<h5>This is an example of how it looks like in HTML</h5>

Defines the sixth and least important heading.

  • h6 means sixth heading, it comes after the fifth heading and its content.

p back to top

Defines a paragraph.

  • Browsers automatically add some space (margin) before and after each <p> element.
  • The margins can be modified with CSS (with the margin properties).

<p>This is an example of how it looks like in HTML</p>

ul back to top

Defines an unordered list.

  • Using the <ul> tag together with the <li> tag creates unordered lists.

<ul>This is an example of how it looks like in HTML</ul>

ol back to top

Defines an ordered list.

  • An ordered list can be numerical or alphabetical.
  • The use of the <li> tag is to define list items within the <ol> tag.

<ol>This is an example of how it looks like in HTML</ol>

li back to top

Defines a list item.

  • The <li> tag is used both in ordered lists (<ol>) and unordered lists (<ul>).

<li>This is an example of how it looks like in HTML</li>

a back to top

Defines a hyperlink, which is used to link from one page to another.

  • The most important attribute of the <a> element is the href attribute, which indicates the link's destination.
  • By default, links will appear as follows in all browsers:
    • An unvisited link is underlined and blue
    • A visited link is underlined and purple
    • An active link is underlined and red

<a>This is an example of how it looks like in HTML</a>

img back to top

Defines an image.

  • The <img> tag has two required attributes: src and alt.
  • Images are not technically inserted into an HTML page, images are linked to HTML pages. The <img> tag creates a holding space for the referenced image.
  • To link an image to another document, simply nest the <img> tag inside <a> tags.

<img>This is an example of how it looks like in HTML</img>

html back to top

Defines the root of an HTML document.

  • The <html> tag tells the browser that this is an HTML document.
  • The <html> tag represents the root of an HTML document.
  • The <html> tag is the container for all other HTML elements (except for the <!DOCTYPE> tag).

<html>This is an example of how it looks like in HTML</html>

head back to top

  • The <head> element is a container for all the head elements.
  • The <head> element can include a title for the document, scripts, styles, meta information, and more.
  • The following elements can go inside the element:
    • title (this element is required in an HTML document)
    • style
    • base
    • link
    • meta
    • script
    • noscript

<head>This is an example of how it looks like in HTML<head>

link back to top

Defines the relationship between a document and an external resource (most used to link to style sheets).

  • The <link> tag is used to link to external style sheets.
  • The <link> element is an empty element, it contains attributes only.
  • This element goes only in the head section, but it can appear any number of times.

<link>This is an example of how it looks like in HTML</link>

title back to top

Defines a title for a document.

  • The <title> tag is required in all HTML documents.
  • The <title> element:
    • defines a title in the browser toolbar
    • provides a title for the page when it is added to favorites
    • displays a title for the page in search-engine results.

<title>This an example of how it looks like in HTML<title>

body back to top

Defines the document's body.

  • The <body> element contains all the contents of an HTML document, such as text, hyperlinks, images, tables, lists, etc.

<body>This an example of how it looks like in HTML<body>

span back to top

Defines a section in a document.

  • The <span> tag provides no visual change by itself.
  • The <span> tag provides a way to add a hook to a part of a text or a part of a document.

<span>This an example of how it looks like in HTML<span>

div back to top

Defines a section in a document.

  • The <div> element is often used as a container for other HTML elements to style them with CSS or to perform certain tasks with JavaScript.
  • The <div> element is very often used together with CSS, to layout a web page.

<div>This an example of how it looks like in HTML<div>

header back to top

Defines a header for a document or section.

  • A header element typically contains:
    • one or more heading elements (h1 - h6)
    • logo or icon
    • authorship information
  • You can have several header elements in one document.
  • A <header> tag cannot be placed within a footer, address or another header element.

<header>This an example of how it looks like in HTML</header>

nav back to top

Defines navigation links.

  • Not all links of a document should be inside a <nav> element. The <nav> element is intended only for major block of navigation links.
  • Browsers, such as screen readers for disabled users, can use this element to determine whether to omit the initial rendering of this content.

<nav>This an example of how it looks like in HTML</nav>

footer back to top

Defines a footer for a document or section.

  • A <footer> element should contain information about its containing element.
  • A <footer> element typically contains:
    • authorship information
    • copyright information
    • contact information
    • sitemap
    • back to top links
    • related documents.
  • You can have several <footer> elements in one document.

<footer>This an example of how it looks like in HTML</footer>

main back to top

Specifies the main content of a document.

  • The content inside the <main> element should be unique to the document. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms.
  • There must not be more than one <main> element in a document. The <main> element must NOT be a descendant of an <article>, <aside>, <footer>, <header>, or <nav> element.

<main>This an example of how it looks like in HTML</main>

section back to top

Defines a section in a document.

  • Chapters, headers, footers, or any other sections of the document are used by this element tag.

<section>This an example of how it looks like in HTML</section>

aside back to top

Defines content aside from the page content.

  • The aside content should be related to the surrounding content.
  • The aside content could be placed as a sidebar in an article.

<aside>This an example of how it looks like in HTML</aside>
